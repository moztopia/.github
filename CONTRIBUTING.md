# 🤝 Contributing to Moztopia

Thank you for your interest in contributing to **Moztopia** projects.  
This document defines how contributions are made, reviewed, and merged.

---

## 📂 Branch Naming

Branches must follow a prefix convention.  
Only the following prefixes are handled by the **automated integration workflow**:

- `feat/<name>` → new features  
- `chore/<name>` → maintenance tasks  
- `bug/<name>` → bug fixes  
- `docs/<name>` → documentation changes  
- `test/<name>` → testing and QA  
- `ai/<name>` → AI framework or automation changes  
- `ci/<name>` → CI/CD pipeline or automation changes  
- `refactor/<name>` → code restructuring without new features  

Other prefixes (e.g., `hotfix/`) require **manual processing** and direct maintainer oversight.

---

## 📝 Commit Guidelines

- Use clear, descriptive commit messages.  
- Reference issues or PRs when applicable.  
- Follow the style rules in `STYLE_GUIDE.md`.  
- Example: `refactor: simplify API response handling`

---

## 🔄 Pull Requests

- All changes must be submitted via Pull Request.  
- PRs must:
  - Pass CI/CD checks  
  - Include relevant documentation updates  
  - Be reviewed by a maintainer (James Hunter or Bruce Anwyl)  
- Draft PRs are encouraged for early feedback.

---

## 🔒 Private vs Public Repos

- **Private repos**: Contributions are limited to Moztopia maintainers.  
- **Public repos**: External contributors may open Issues or PRs.  
- Sensitive discussions and fixes remain private.
