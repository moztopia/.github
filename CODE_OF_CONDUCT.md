# 📜 Code of Conduct

This Code of Conduct applies to all **Moztopia** repositories and community spaces.  
It defines expected behavior, unacceptable conduct, and enforcement procedures.

---

## ✅ Expected Behavior

- Treat all contributors with respect and professionalism.  
- Provide constructive feedback in code reviews and discussions.  
- Follow established workflows (branching, commits, pull requests).  
- Protect confidentiality of private repositories and internal discussions.  
- Uphold security practices defined in `SECURITY.md`.

---

## 🚫 Unacceptable Behavior

- Harassment, discrimination, or personal attacks.  
- Public disclosure of private Moztopia information.  
- Introducing malicious code, backdoors, or unsafe dependencies.  
- Ignoring branch protections, CI/CD checks, or CODEOWNERS rules.  
- Circumventing governance documents (README, CONTRIBUTING, STYLE_GUIDE, SECURITY, SUPPORT).

---

## 🛠️ Enforcement

- All violations will be reviewed by maintainers (James Hunter and Bruce Anwyl).  
- Sanctions may include:
  - Revocation of repository access  
  - Rejection of pull requests  
  - Removal from organization membership  
- Enforcement decisions are final and documented internally.

---

## 📌 Scope

This Code of Conduct applies to:

- All Moztopia repositories  
- GitHub issues, pull requests, and discussions  
- Internal communication channels related to Moztopia projects
